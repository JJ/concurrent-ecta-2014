Functional and concurrent languages are just different

  • Concurrency: Actors, agents, messages.
  • Functions, lists.

Scala, Erlang and Clojure are multi-paradigm languages.

Actors: concurrent units of execution

  • Implemented as processes in the Erlang VM.
  • Lightweight life cycle.
  • Using immutable messages.

Image from Happenningfish

Functions as the main concept

  • Efficient implementation of lists.
  • Use of high order functions.
  • Macro system.

Image from NASA Goddard

Using a pool-based algorithm

  • A pool is an actor.
  • The client is an actor requesting individuals from the pool and returning them.
  • Separation of architecture and problem encoding; fitness, operators, selection criteria, replacement policy: injected into the pool.

What gives?

  • Disappointing results.
  • Intra-actor concurrency rather low: pool communications in a queue.
  • Different models shoud be tried: persistent pool, non-Erlang pool...

Worth the while?

  • Nice introduction to a non-standard language.
  • But more of a stepping-stone to other languages: Scala, Go.
  • Combination with asynchronous event processing.
